# Colab and CloudStudio A100 Runner 2026-06-16

This package records the resumable A100 experiment runner prepared for the
2026-06-16 benchmark pass.

## CloudStudio

Clone the PR branch, switch the workspace to an A100 GPU, then run:

```bash
python scripts/run_colab_a100_experiment.py \
  --output-root reports/colab_a100_20260616
```

The script uses the current checkout by default. It writes per-branch artifacts
under `reports/colab_a100_20260616/runs/`, including benchmark JSON/Markdown,
branch logs, and `gpu_smi.csv`.

The final outputs are:

```text
reports/colab_a100_20260616/colab_a100_benchmark_report_2026-06-16.md
reports/colab_a100_20260616/colab_a100_results_2026-06-16.zip
```

If the workspace disconnects, rerun the same command. Completed branches are
detected from their saved benchmark JSON and GPU CSV.

## Google Colab

Open `notebooks/colab_a100_bootstrap_20260616.ipynb` in an A100 runtime and run
the single code cell. Outputs are saved to:

```text
/content/drive/MyDrive/nano-vllm-colab-a100/2026-06-16/
```

## Notes

- If `nvidia-smi` does not report an A100, the runner only performs the smoke
  branch and does not interpret A100 performance.
- This runner does not implement new SSE flush or admission-control behavior.
  The `server_admission_*` and `server_stream_flush_*` columns are passive
  report fields; blank values mean the current server did not emit those
  metrics.
- The source-transfer patch and metadata are kept under
  `artifacts/colab_a100_20260616/` for reproducibility.
